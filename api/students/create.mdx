---
title: 'Create Student'
openapi: 'POST /students'
---

Creates a new student record in the school.

## Request Body

<ParamField body="document_type" type="string" required>
  Type of identification document. Valid values: `TI` (Tarjeta de Identidad), `RC` (Registro Civil), `CC` (Cédula de Ciudadanía), `CE` (Cédula de Extranjería), `NUIP` (Número Único de Identificación Personal), `PASSPORT`
</ParamField>

<ParamField body="document_number" type="string" required>
  Identification document number. Spaces and dots are automatically removed
</ParamField>

<ParamField body="full_name" type="string" required>
  Student's full name
</ParamField>

<ParamField body="user_id" type="uuid" optional>
  Link to an existing user account (if the student has a user account)
</ParamField>

<ParamField body="birth_date" type="date" optional>
  Date of birth in YYYY-MM-DD format
</ParamField>

<ParamField body="gender" type="string" optional>
  Student's gender
</ParamField>

<ParamField body="extra_data" type="object" optional>
  Additional custom data for the student. Must be a JSON object
</ParamField>

<ParamField body="piar_data" type="object" optional>
  PIAR (Individual Plan for Reasonable Adjustments) data for students with special educational needs. Must be a JSON object
</ParamField>

<ParamField body="guardians" type="array" optional>
  Array of guardian objects to associate with this student
  
  <Expandable title="Guardian Object">
    <ParamField body="document_type" type="string" required>
      Guardian's identification document type
    </ParamField>
    
    <ParamField body="document_number" type="string" required>
      Guardian's identification document number
    </ParamField>
    
    <ParamField body="full_name" type="string" required>
      Guardian's full name
    </ParamField>
    
    <ParamField body="relationship" type="string" required>
      Relationship to the student (e.g., "Father", "Mother", "Uncle", "Grandmother")
    </ParamField>
    
    <ParamField body="phone" type="string" optional>
      Contact phone number
    </ParamField>
    
    <ParamField body="email" type="string" optional>
      Contact email address
    </ParamField>
    
    <ParamField body="address" type="string" optional>
      Physical address
    </ParamField>
    
    <ParamField body="occupation" type="string" optional>
      Guardian's occupation
    </ParamField>
    
    <ParamField body="workplace" type="string" optional>
      Guardian's workplace
    </ParamField>
    
    <ParamField body="is_primary" type="boolean" default="false">
      Whether this is the primary guardian
    </ParamField>
    
    <ParamField body="priority" type="integer" default="1">
      Contact priority order (minimum: 1). Lower numbers indicate higher priority
    </ParamField>
    
    <ParamField body="can_pickup" type="boolean" default="false">
      Whether this guardian is authorized to pick up the student
    </ParamField>
    
    <ParamField body="is_emergency_contact" type="boolean" default="false">
      Whether this guardian should be contacted in emergencies
    </ParamField>
    
    <ParamField body="is_active" type="boolean" default="true">
      Whether the guardian record is active
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns the created student object with associated guardians.

<ResponseField name="id" type="uuid">
  Unique identifier for the created student
</ResponseField>

<ResponseField name="school_id" type="uuid">
  School identifier
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Will always be `true` for newly created students
</ResponseField>

See the [List Students](/api/students/list) endpoint for the complete response schema.

## Validation Rules

- The combination of `document_type` and `document_number` must be unique within the school
- `document_type` must be one of the allowed values
- `document_number` cannot be empty after normalization (removing spaces and dots)
- If guardians are provided with the same `document_type` and `document_number` as an existing guardian, the existing guardian record will be updated and linked to the student

## Permissions Required

One of the following permissions:
- `write:enrollment`
- `write:all`

import requests

response = requests.post(
    "https://api.athena-erp.com/students",
    headers={
        "Authorization": "Bearer YOUR_API_TOKEN",
        "Content-Type": "application/json"
    },
    json={
        "document_type": "TI",
        "document_number": "1234567890",
        "full_name": "Juan Pérez García",
        "birth_date": "2010-05-15",
        "gender": "M",
        "guardians": [
            {
                "document_type": "CC",
                "document_number": "98765432",
                "full_name": "María García",
                "relationship": "Mother",
                "phone": "+57 300 1234567",
                "email": "maria.garcia@example.com",
                "is_primary": True,
                "priority": 1,
                "can_pickup": True,
                "is_emergency_contact": True
            }
        ]
    }
)

student = response.json()
print(f"Created student: {student['full_name']} (ID: {student['id']})")
