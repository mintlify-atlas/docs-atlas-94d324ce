## Get Current School

<api-endpoint method="GET" path="/schools/me" />

Retrieve information about the current school (tenant) based on the authenticated user's context.

### Response

<response-field name="id" type="uuid">
  Unique identifier for the school
</response-field>

<response-field name="name" type="string">
  Official name of the school
</response-field>

<response-field name="nit" type="string">
  Tax identification number (NIT) for the school
</response-field>

<response-field name="resolution" type="string">
  Official resolution or authorization number from the education ministry
</response-field>

<response-field name="is_active" type="boolean">
  Whether the school is currently active in the system
</response-field>

<response-field name="pei_summary" type="string">
  Summary of the school's Institutional Educational Project (PEI)
</response-field>

<response-field name="settings" type="object">
  School configuration settings including SIMAT integration, branding, security, and habeas data policies
</response-field>

<response-field name="created_at" type="datetime">
  When the school was created in the system
</response-field>

<response-field name="updated_at" type="datetime">
  Last update timestamp
</response-field>

<request-example>
```bash
curl -X GET "https://api.athena-erp.com/schools/me" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-School-ID: your-school-id"
```
</request-example>

## Get School by ID (Admin)

<api-endpoint method="GET" path="/admin/schools/{school_id}" />

Retrieve detailed information about a specific school. Requires super-admin permissions.

### Path Parameters

<param-field path="school_id" type="uuid" required>
  Unique identifier of the school
</param-field>

<request-example>
```bash
curl -X GET "https://api.athena-erp.com/admin/schools/987fcdeb-51a2-43d1-b789-123456789abc" \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```
</request-example>

## School Settings

The `settings` object contains the following configuration options:

### SIMAT Settings

<response-field name="settings.simat_settings" type="object">
  Integration with Colombian SIMAT (Student Information Management System)
  - `enabled` (boolean) - Whether SIMAT integration is active
  - `institution_code` (string) - Official SIMAT institution code
  - Additional SIMAT configuration parameters
</response-field>

### Branding Settings

<response-field name="settings.branding_settings" type="object">
  Visual branding and customization
  - `primary_color` (string) - Primary brand color (hex format)
  - `secondary_color` (string) - Secondary brand color
  - `logo_r2_object_key` (string) - Storage key for school logo
  - `files` (array) - Additional uploaded media files
</response-field>

### Security Settings

<response-field name="settings.security_settings" type="object">
  Security and access control policies
  - `require_2fa` (boolean) - Require two-factor authentication
  - `session_timeout_minutes` (integer) - Session timeout duration
  - `password_policy` (object) - Password requirements
  - Additional security policies
</response-field>

### Habeas Data

<response-field name="settings.habeas_data_text" type="string">
  Official text for personal data protection (Habeas Data) consent, per Colombian Law 1581 of 2012
</response-field>

## Permissions

- **Current School** (`/schools/me`): Requires any authenticated user
- **Admin Endpoints** (`/admin/schools/*`): Requires `manage:schools` permission

## Notes

<info>
**Tenant Context**: Most endpoints in the API operate within the context of a single school (tenant). The `X-School-ID` header or user's school membership determines which school's data is accessed.
</info>

<tip>
**Settings Auto-Creation**: School settings are automatically created when a school is created. If settings don't exist for some reason, they will be created on first access.
</tip>
