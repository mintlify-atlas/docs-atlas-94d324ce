title: "Grades"
description: "Manage evaluation activities and student scores"
---

The grades API allows you to create evaluation activities (tasks, exams, projects) and record student scores for those activities.

## Evaluation Activities

Evaluation activities are assessments that students complete for a specific course offering. They can be weighted and have different types.

### Create Evaluation Activity

<ParamField body="course_offering_id" type="uuid" required>
  ID of the course offering this activity belongs to
</ParamField>

<ParamField body="teacher_assignment_id" type="uuid" required>
  ID of the teacher assignment who created this activity
</ParamField>

<ParamField body="academic_period_id" type="uuid" required>
  ID of the academic period (quarter, semester, etc.)
</ParamField>

<ParamField body="title" type="string" required>
  Title of the evaluation activity
</ParamField>

<ParamField body="activity_type" type="string" required>
  Type of activity: `task`, `quiz`, `exam`, `workshop`, `project`, or `other`
</ParamField>

<ParamField body="weight" type="float" required>
  Weight of this activity in the final grade (0-100)
</ParamField>

<ParamField body="is_gradable" type="boolean" default="true">
  Whether this activity affects the student's grade
</ParamField>

<ParamField body="due_at" type="datetime">
  Due date and time for the activity
</ParamField>

<ParamField body="attachments" type="array">
  Array of attachment objects (files, links, etc.)
</ParamField>

<ResponseField name="id" type="uuid">
  Unique identifier for the activity
</ResponseField>

<ResponseField name="school_id" type="uuid">
  ID of the school
</ResponseField>

<ResponseField name="course_offering_id" type="uuid">
  ID of the course offering
</ResponseField>

<ResponseField name="teacher_assignment_id" type="uuid">
  ID of the teacher assignment
</ResponseField>

<ResponseField name="academic_period_id" type="uuid">
  ID of the academic period
</ResponseField>

<ResponseField name="title" type="string">
  Title of the activity
</ResponseField>

<ResponseField name="activity_type" type="string">
  Type of activity
</ResponseField>

<ResponseField name="weight" type="float">
  Weight in final grade (0-100)
</ResponseField>

<ResponseField name="is_gradable" type="boolean">
  Whether this activity is gradable
</ResponseField>

<ResponseField name="due_at" type="datetime">
  Due date and time
</ResponseField>

<ResponseField name="attachments" type="array">
  Array of attachments
</ResponseField>

<ResponseField name="created_at" type="datetime">
  Creation timestamp
</ResponseField>

<ResponseField name="updated_at" type="datetime">
  Last update timestamp
</ResponseField>

## Activity Scores

Activity scores represent the grades students receive for specific evaluation activities.

### Create Activity Score

<ParamField body="evaluation_activity_id" type="uuid" required>
  ID of the evaluation activity
</ParamField>

<ParamField body="course_enrollment_id" type="uuid" required>
  ID of the course enrollment
</ParamField>

<ParamField body="student_id" type="uuid" required>
  ID of the student
</ParamField>

<ParamField body="score" type="float" required>
  Score value (0.0 to 5.0)
</ParamField>

<ParamField body="observations" type="string">
  Optional observations or comments about the score
</ParamField>

<ResponseField name="id" type="uuid">
  Unique identifier for the score record
</ResponseField>

<ResponseField name="school_id" type="uuid">
  ID of the school
</ResponseField>

<ResponseField name="evaluation_activity_id" type="uuid">

</ResponseField>

<ResponseField name="course_enrollment_id" type="uuid">
  ID of the course enrollment
</ResponseField>

<ResponseField name="student_id" type="uuid">
  ID of the student
</ResponseField>

<ResponseField name="score" type="float">
  Score value (0.0 to 5.0)
</ResponseField>

<ResponseField name="observations" type="string">
  Observations or comments
</ResponseField>

<ResponseField name="recorded_by" type="uuid">
  ID of the user who recorded the score

## Score Scale

Scores in Athena ERP use a scale from 0.0 to 5.0, where:
- **5.0**: Outstanding
- **4.0-4.9**: Excellent
- **3.0-3.9**: Good
- **2.0-2.9**: Acceptable
- **0.0-1.9**: Needs improvement

## Permissions

- **Create activities**: Requires `write:activities`, `write:grades`, or `write:all` permission
- **List activities**: Requires `read:grades` or `read:all` permission
- **Create scores**: Requires `write:grades` or `write:all` permission
- **List scores**: Requires `read:grades` or `read:all` permission
