---
title: "Update Enrollment"
api: "PATCH /api/enrollments/{enrollment_id}"
description: "Update an existing enrollment's information"
---

## Path Parameters

<ParamField path="enrollment_id" type="uuid" required>
  Unique identifier of the enrollment to update
</ParamField>

## Request Body

All fields are optional. Only include fields you want to update.

<ParamField body="grade_level" type="string">
  Update the grade level
</ParamField>

<ParamField body="group_code" type="string">
  Update the group or class code
</ParamField>

<ParamField body="shift" type="string">
  Update the school shift
</ParamField>

<ParamField body="status" type="string">
  Update the enrollment status. When changed, this will be automatically tracked in the status_history array.
  
  Common status values:
  - `pending_documents` - Awaiting document submission
  - `active` - Enrollment is active
  - `completed` - Enrollment completed
  - `withdrawn` - Student withdrawn
</ParamField>

<ParamField body="status_history" type="array">
  Manually update the status history array (advanced use only)
</ParamField>

<ParamField body="simat_status" type="object">
  Update SIMAT (Colombian education system) status data
</ParamField>

<ParamField body="extra_data" type="object">
  Update additional custom data for the enrollment
</ParamField>

## Status Workflow

The enrollment status typically follows this workflow:

1. **pending_documents** - Initial state when enrollment is created
2. **active** - All documents uploaded and enrollment is active
3. **completed** - School year completed
4. **withdrawn** - Student withdrawn from enrollment

When the status is changed, the system automatically:
- Appends the change to the `status_history` array
- Records the timestamp of the change
- Records the user ID who made the change

## Permissions Required

- `write:enrollment`
- `write:all`
