## List Discipline Cases

<api-endpoint method="GET" path="/discipline/cases" />

Retrieve all discipline cases for the current school with optional filtering.

### Query Parameters

<param-field path="phase" type="string" optional>
  Filter by case phase (e.g., "incident", "investigation", "resolution")
</param-field>

<param-field path="severity" type="string" optional>
  Filter by severity level (e.g., "low", "medium", "high", "critical")
</param-field>

<param-field path="search" type="string" optional>
  Search term to filter by description or location
</param-field>

### Response

<response-field name="id" type="uuid">
  Unique identifier for the discipline case
</response-field>

<response-field name="school_id" type="uuid">
  ID of the school this case belongs to
</response-field>

<response-field name="reported_by" type="uuid">
  User ID of the person who reported the incident
</response-field>

<response-field name="incident_date" type="date">
  Date when the incident occurred
</response-field>

<response-field name="location" type="string">
  Physical location where the incident took place
</response-field>

<response-field name="description" type="string">
  Detailed description of the incident
</response-field>

<response-field name="category" type="string">
  Category of the incident (e.g., "bullying", "violence", "disrespect")
</response-field>

<response-field name="severity" type="string">
  Severity level of the incident
</response-field>

<response-field name="phase" type="string">
  Current phase of the case (incident, investigation, resolution, closed)
</response-field>

<response-field name="resolution_type" type="string">
  Type of resolution applied (if resolved)
</response-field>

<response-field name="resolution_notes" type="string">
  Notes about the resolution
</response-field>

<response-field name="escalated_at" type="datetime">
  Timestamp when the case was escalated
</response-field>

<response-field name="closed_at" type="datetime">
  Timestamp when the case was closed
</response-field>

<response-field name="participants" type="array">
  List of participants involved in the case (students, staff, guardians)
</response-field>

<response-field name="events" type="array">
  Timeline of events related to the case
</response-field>

<response-field name="created_at" type="datetime">
  Timestamp when the case was created
</response-field>

<response-field name="updated_at" type="datetime">
  Timestamp of the last update
</response-field>

```bash
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-School-ID: your-school-id"
```

## Create Discipline Case

<api-endpoint method="POST" path="/discipline/cases" />

Create a new discipline case with optional participants and initial event.

### Request Body

<param-field path="incident_date" type="date" required>
  Date when the incident occurred
</param-field>

<param-field path="location" type="string" optional>
  Physical location of the incident
</param-field>

<param-field path="description" type="string" required>
  Detailed description of the incident
</param-field>

<param-field path="category" type="string" required>
  Category of the incident
</param-field>

<param-field path="severity" type="string" required>
  Severity level (low, medium, high, critical)
</param-field>

<param-field path="phase" type="string" default="incident">
  Initial phase of the case
</param-field>

<param-field path="resolution_type" type="string" optional>
  Type of resolution (if applicable)
</param-field>

<param-field path="resolution_notes" type="string" optional>
  Resolution notes
</param-field>

<param-field path="participants" type="array" optional>
  Array of participants (students, staff, guardians) involved in the case
</param-field>

<param-field path="initial_event" type="object" optional>
  Initial event to add to the case timeline
</param-field>

```bash
curl -X POST "https://api.athena-erp.com/discipline/cases" \

  -H "X-School-ID: your-school-id" \
```

## Update Discipline Case

<api-endpoint method="PATCH" path="/discipline/cases/{case_id}" />

Update an existing discipline case. Only provided fields will be updated.

### Path Parameters

<param-field path="case_id" type="uuid" required>
  Unique identifier of the discipline case
</param-field>

### Request Body

All fields are optional. Only include fields you want to update.

<param-field path="incident_date" type="date" optional>
  Date when the incident occurred
</param-field>

<param-field path="location" type="string" optional>
  Physical location of the incident
</param-field>

<param-field path="description" type="string" optional>
  Description of the incident
</param-field>

<param-field path="category" type="string" optional>
  Category of the incident
</param-field>

<param-field path="severity" type="string" optional>
  Severity level
</param-field>

<param-field path="phase" type="string" optional>
  Current phase (incident, investigation, resolution, closed)
</param-field>

<param-field path="resolution_type" type="string" optional>
  Type of resolution
</param-field>

<param-field path="resolution_notes" type="string" optional>
  Resolution notes
</param-field>

<param-field path="escalated_at" type="datetime" optional>
  Escalation timestamp
</param-field>

<param-field path="closed_at" type="datetime" optional>
  Case closure timestamp
</param-field>

<request-example>
```bash
curl -X PATCH "https://api.athena-erp.com/discipline/cases/123e4567-e89b-12d3-a456-426614174000" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-School-ID: your-school-id" \
  -H "Content-Type: application/json" \
  -d '{
    "phase": "resolution",
    "resolution_type": "warning",
    "resolution_notes": "Se aplicó amonestación verbal y se notificó a los padres"
  }'
```
</request-example>

## Add Participant

<api-endpoint method="POST" path="/discipline/cases/{case_id}/participants" />

Add a participant (student, staff member, or guardian) to an existing discipline case.

### Path Parameters

<param-field path="case_id" type="uuid" required>
  Unique identifier of the discipline case
</param-field>

### Request Body

<param-field path="student_id" type="uuid" optional>
  Student ID (provide one of student_id, user_id, or guardian_id)
</param-field>

<param-field path="user_id" type="uuid" optional>
  Staff user ID
</param-field>

<param-field path="guardian_id" type="uuid" optional>
  Guardian ID
</param-field>

<param-field path="role_in_case" type="string" required>
  Role in the case (e.g., "involved", "witness", "reporter", "mediator")
</param-field>

<request-example>
```bash
curl -X POST "https://api.athena-erp.com/discipline/cases/123e4567-e89b-12d3-a456-426614174000/participants" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-School-ID: your-school-id" \
  -H "Content-Type: application/json" \
  -d '{
    "student_id": "222bcd33-eff4-5678-9012-345678901efg",
    "role_in_case": "witness"
```

## Permissions

All discipline case endpoints require one of the following permissions:
- `write:convivencia` - For creating and modifying cases
- `write:due_process` - For managing due process procedures
- `read:all` - For viewing cases (read-only)
- `write:all` - Full access to all operations
